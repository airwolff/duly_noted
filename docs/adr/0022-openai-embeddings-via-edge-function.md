# 0022 — OpenAI text-embedding-3-small via Edge Function for query embedding

- Status: Accepted
- Date: 2026-05-11
- Slice: 6

## Context

ADR 0021 commits to a pgvector-backed semantic search arm. That decision implies (a) an embedding model and (b) a runtime surface where query-time embedding generation happens. Both decisions have multiple viable answers; this ADR records both because they are tightly coupled.

The semantic arm requires:

- **Index-time embeddings** — one per segment, generated once when a meeting enters the `embedding` pipeline stage. Lives on the worker, which already holds vendor API keys per CLAUDE.md §6.
- **Query-time embeddings** — one per user search query, generated on demand. The search RPC takes the embedding as a parameter, so the embedding has to be computed before the RPC call.

CLAUDE.md §6 prohibits third-party model vendor API keys in `apps/web`. The query-time embedding therefore cannot happen in the web app's request handlers (server components, route handlers, or anywhere else under Cloudflare Pages).

The remaining surfaces that can hold the embedding key:

- **`apps/worker`** — holds vendor keys; can run code on demand but it polls Postgres on a long-running loop. A new web→worker→web round trip for query-time embedding is the wrong shape.
- **`supabase/functions/`** — Deno Edge Functions hold vendor keys per the existing pattern (`asr-webhook` precedent). Short-lived, request/response, JWT-aware.

Embedding-model choice:

- **OpenAI `text-embedding-3-small`** — 1536-dim default, $0.02/1M input tokens (Standard), 8,192-token context. Most widely integrated; cheapest commercial option; reliable.
- **OpenAI `text-embedding-3-large`** — 3,072-dim, $0.13/1M tokens, ~1-2 MTEB points better than `-small`. Six times the cost for marginal quality.
- **Self-hosted open-source** (`all-MiniLM-L6-v2`, `nomic-embed-text-v1`) — $0 in API cost; needs Render compute for inference; adds a service to ops.
- **Other commercial embedding APIs** (Cohere, Voyage AI, Jina) — comparable to or slightly better than OpenAI on benchmarks at similar prices. Adding a new vendor relationship for marginal benefit.

Anthropic does not offer an embeddings API, so consolidating on the existing LLM vendor is not an option.

## Considered options

- **OpenAI `text-embedding-3-small` + Edge Function `search` (chosen).** Index time on the worker; query time in the Edge Function. Single new vendor key.
- **OpenAI `text-embedding-3-large`.** Quality delta does not justify the 6× cost increase for a v1 newsroom search workload where retrieval is already gated by RLS scope.
- **Self-hosted open-source embeddings (S3 from the search-implementation KB).** Saves recurring vendor cost. Costs Render compute + a model server. Operational surface added to save dollars that, at v1 scale, are pennies.
- **OpenAI embeddings via web-app server-component (with key in Cloudflare env).** Violates CLAUDE.md §6 vendor-key isolation. Out.
- **Two-round-trip pattern: Edge Function returns embedding, web app calls `search_segments` RPC separately.** More idiomatic to existing Supabase-client data-fetching patterns. Doubles browser round trips. Exposes the embedding to the client unnecessarily. JWT enforcement split across two gates.

## Decision

- **Index-time embeddings.** Generated by `apps/worker/src/embedding/` via OpenAI `text-embedding-3-small` (1536-dim, native). One embedding per segment. Worker holds `OPENAI_API_KEY`.
- **Query-time embeddings.** Generated by Edge Function `supabase/functions/search`. Receives `{ query, match_count? }`, validates the caller's JWT, generates the embedding, calls `search_segments` RPC server-side, returns the result set to the web layer. The Edge Function holds `OPENAI_API_KEY`.
- **Web app holds neither.** Cloudflare Pages env does not include `OPENAI_API_KEY`.

The Edge Function does NOT disable JWT verification (distinct from `asr-webhook`, which is a vendor callback). The endpoint is an authenticated user-facing API; the caller's JWT is required.

Native 1536 dimensions per ADR 0021 — no Matryoshka truncation at v1.

## Consequences

**Accepted:**

- Single new vendor key, isolated to the surfaces that need it per the existing CLAUDE.md §6 pattern.
- Query latency: one Edge Function invocation per search query (~50-200ms cold start + OpenAI call ~50-100ms + RPC call ~10-50ms). Total search-result latency dominated by OpenAI's response time, not Supabase or Cloudflare.
- Cost at v1 scale is negligible. Index time: <$0.01 backfill + ~1 cent/week per board ongoing. Query time: ~$0.000002 per query at typical query length.
- The two-surface key isolation (worker holds for index, Edge Function holds for query) follows the established `ASR_VENDOR_API_KEY` pattern verbatim.

**Risks:**

- OpenAI downtime breaks search. The reader continues to function (browse-only via list pages); search returns the error state. Acceptable degradation.
- 8,192-token input limit per call. v1 segment lengths are well below this; the worker handler validates length defensively before each call.
- Embedding-model deprecation. OpenAI has retained `text-embedding-ada-002` after the v3 launch, suggesting deprecation cadence is slow. If `text-embedding-3-small` is deprecated, re-embed via a new model is a one-shot script invocation against the cloud DB. Not a v1 concern.

**Revisit trigger:**

- OpenAI pricing changes materially (e.g., 10×).
- A different embedding model produces meaningfully better newsroom-search quality on the project's corpus and the cost differential is justifiable.
- Self-hosting becomes attractive (e.g., the project moves off Cloudflare Pages and gains compute headroom for free-tier inference).

## Alternatives considered

- **`text-embedding-3-large`.** Rejected: 6× cost for marginal quality at v1 scale.
- **Self-hosted embeddings (S3 from the search KB).** Rejected: operational surface for pennies of savings.
- **Cohere / Voyage / Jina.** Rejected: new vendor relationship for no decisive benefit. May reconsider if newsroom-search quality on `text-embedding-3-small` proves insufficient after Slice 6 ships.
- **Query embedding inside the web app's server handler.** Rejected: violates CLAUDE.md §6 vendor-key isolation. Cloudflare Pages env is a public-key-only surface for this project.
- **Two-round-trip pattern (Edge Function embeds only, web calls RPC separately).** Rejected: doubles round trips, exposes embedding to client, splits JWT enforcement.
